《入侵报警系统工程设计规范》学习笔记

规范编号：GB50394-2007

关键词

- IAS:入侵报警系统
- fault condition：故障状态
- tamper alarm：防拆报警
- set condition：设防
- unset condition：撤防
- perimeter：周界
- leakage alarm：漏报警
- false alarm : 误报警
- emergency alarm : 紧急报警
- response time : 报警响应时间

强条（5条）

1.入侵报警系统中使用的设备必须符合国家法律法规和现行强制性标准的要求，并经法定机构检验或认证合格。（3.0.3）

2.入侵报警系统不得有漏报警。（5.2.2）

3.入侵报警功能设计应符合下列规定：(5.2.3)

紧急报警装置应设置为不可撤防状态，应有防误触发措施，被触发后应自锁。
当下列任何情况发生时，报警控制设备应发出声、光报警信息，报警信息应能保持到手动复位，报警信号应无丢失：

在设防状态下，当探测器探测到有入侵发生或触动紧急报警装置时，报警控制设备应显示出报警发生的区域或地址；
在设防状态下，当多路探测器同时报警(含紧急报警装置报警)时，报警控制设备应依次显示出报警发生的区域或地址。
报警发生后，系统应能手动复位，不应自动复位。
在撤防状态下，系统不应对探测器的报警状态做出响应。

4.防破坏及故障报警功能设计应符合下列规定：(5.2.4)
当下列任何情况发生时，报警控制设备上应发出声、光报警信息，报警信息应能保持到手动复位，报警信号应无丢失：

在设防或撤防状态下，当入侵探测器机壳被打开时。
在设防或撤防状态下，当报警控制器机盖被打开时。
在有线传输系统中，当报警信号传输线被断路、短路时。
在有线传输系统中，当探测器电源线被切断时。
当报警控制器主电源／备用电源发生故障时。
在利用公共网络传输报警信号的系统中，当网络传输发生故障或信息连续阻塞超过30s时。

5.系统供电暂时中断，恢复供电后，系统应不需设置即能恢复原有工作状态。(9.0.1-3)

相关规范

GB50348《安全防范工程技术规范》
GA/T 368《入侵报警系统技术要求》

建设单位设计要求

  • 设计任务书
  • 建筑平面图，现场勘察报告
  • 防区的划分
  • 明确探测器，传输设备的位置和选型
  • 有线传输为主，无线传输为辅

系统组成

  • 前端设备
  • 传输设备
  • 处理/控制/管理设备
  • 显示/记录设备

按传输方式划分：

分线制：报警时间不大于2S，防区少（小于16路），前端与控制系统距离小于100m；宜采用不小于5芯通信电缆，线芯不小于0.5平方毫米；
总线制：报警时间不大于2S，防区多，前端与控制系统距离小于1500m;宜采用不少于6芯通信电缆，线芯不小于1.0平方毫米；
无线制：报警时间不大于2S；
公共网络：电话报警时间不大于20S；

防护体系设计：

纵深防护：周界，监视区，防护区，禁区；
局部防护

探测器类型

警戒范围 探测器类型
空间式 室内多普勒微波探测器、室内用被动红外探测器、室内用超声波多普勒微波探测器、微波和被动红外复合入侵探测器、磁开关入侵探测器、室内用被动式玻璃破碎探测器等
面控式 振动入侵探测器、声控振动双技术玻璃破碎探测器等
线控式 主动式红外入侵探测器、微波墙式探测器、激光式探测器、光纤式周界探测器、长导电体断裂原理探测器、振动电缆探测器、泄漏电缆探测器、电场线感应式探测器等
开关点控式 磁开关入侵探测器、微动开关、紧急报警装置、压力垫、短导电体的断裂原理探测器等

安装工艺要求

  • 无线报警应有欠压报警；电池有效时间不少于6个月，欠压报警状态可工作7天
  • 每个探测器设置为一个防区
  • 周界独立防区长度不宜大于200m
  • 每个独立防区的紧急报警数量不应大于4个
  • 系统软件异常后，应能在3S内发出故障报警
  • 距离远，电磁环境恶劣，选用光缆
  • 报警信号线与220V交流电源线分开敷设
  • 引出地面的出线口，宜选在相对隐蔽地点，并宜在出口处设置，从地面高度不低于3米的出线防护钢管，且周围5米内不应有易攀登的物体
  • 报警系统工作不小于8小时
  • 室外报警设备应防雷
  • 监控中心门、窗应设置防护措施
  • 管线敷设图应标明管线的敷设安装方式、型号、路由、数量、末端出线盒的位置、高度等，分线箱应根据需要标明线缆的走向、端子号，并根据要求在主干线上预留适当数量的备用线缆，并列出材料统计表
